# Future GitHub Issues

Use these as initial GitHub issues once the public repository is created.

## Grocery MVP

### Setup wizard

Collect household name, ZIP/city area, radius, shopping days, favorite stores, budget mode, and dietary preferences.

### Chat command parser

Turn natural messages like "add eggs", "remove bananas", and "what do we need?" into structured grocery actions.

### SQLite storage backend

Add a durable multi-household backend while keeping JSON useful for local development.

### Daily deal scan job

Read current grocery list, query configured deal sources, rank matches, and send a digest.

### Weekly shopping reminder

On configured shopping days, send a list summary grouped by category and highlight best deals.

## Integrations

### iMessage adapter via BlueBubbles

Research and implement a messaging adapter for iMessage group chats through BlueBubbles.

### Apple shared list backend research

Evaluate Apple Notes, Apple Reminders, CalDAV, Shortcuts, and macOS automation for shared iPhone list syncing.

### Store deal adapters

Create adapters for Kroger, Walmart, Target, Costco, Aldi, and weekly ad sources where terms/APIs permit.

## Expansion modules

### Recipe module

Suggest easy dinners/baking recipes and add selected ingredients after confirmation.

### Activities/date-night module

Find local activities by area, radius, budget, date, and weather.

## Commerce experiments

### Instacart/cart research

Explore cart creation and purchase limitations. Do not enable real orders until explicit approval and safety controls exist.

### AgentCash purchase flow

Prototype payment authorization with strict confirmation, spend caps, audit logs, and test/sandbox paths before any real purchase.
